What is deference between HTML and XHTML

HTML has a more forgiving syntax compared to XHTML. HTML allows certain errors and inconsistencies, such as unclosed tags or attribute values without quotes, and still renders the page. On the other hand


HTML is widely supported by all modern web browsers, including older versions.

XHTML :


HTML allows certain errors and inconsistencies, such as unclosed tags or attribute values without quotes, and still renders the page. On the other hand, XHTML follows a stricter XML-based syntax and requires well-formedness. All tags must be properly closed, attribute values must be enclosed in quotes, and lowercase tag names are mandatory.
